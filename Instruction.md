# Инструкция по работе с репозиторием

Этот репозиторий используется для выполнения учебных заданий по практике. Каждый студент работает со своей копией репозитория, оформляет каждое задание отдельной веткой и отправляет результат через pull request.

## Порядок работы

1. Зарегистрируйтесь на GitHub, если у вас еще нет аккаунта.
2. Сделайте fork этого репозитория в свой аккаунт.
3. Склонируйте свой fork на компьютер.
4. Создайте и используйте две ветки:
	- `main` — основная ветка с итоговым кодом.
	- `dev` — рабочая ветка, в которой выполняются задания.
5. Для каждого задания переходите в ветку `dev`, вносите изменения и проверяйте результат локально.
6. После завершения задания отправляйте изменения в свой удаленный репозиторий и создавайте pull request из `dev` в `main`.
7. В pull request обязательно назначайте [меня](https://github.com/elizaveta-rybina) в reviewers для проверки работы. Для этого зайти в Settings (Настройки) -> Collaborators (Соавторы). Нажать Add people (Добавить людей) и выбрать [меня](https://github.com/elizaveta-rybina) в качестве соавтора. Дальше принимается приглашение и можно будет уже выбрать в качестве ревьювера.
8. После замечаний при необходимости дорабатывайте код в `dev` и обновляйте тот же pull request.
9. После проверки и одобрения pull request может быть добавлен в `main`.

## Важные требования

- Каждое задание оформляется отдельным pull request.
- Коммиты должны быть осмысленными.
- Рабочие изменения делаются только в `dev`.
- В `main` должен попадать только проверенный и готовый код.
- Перед отправкой pull request убедитесь, что код запускается и не содержит ошибок.
